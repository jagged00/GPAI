Confederations Types

We're getting confederation types. Old: create_confederation = { name = key/dynamic description } New: create_confederation - Creates a confederation and adds the scoped character as a member. The confederation will be saved to scope:new_confederation usage: create_confederation = { name = key/dynamic description type = <confederation type> # Optional. Type of confederation to create. leader = <house> # Optional. House that leads the confederation. } Supported Scopes: character Targets: house_confederation - Scope to the house's confederation Input Scopes: dynasty_house Output Scopes: confederation leading_house - Scope to the confederation's leading dynasty house Input Scopes: confederation Output Scopes: dynasty_house confederation_type - Get the confederation_type with the specified key Requires Data: yes Global Link: yes Output Scopes: confederation_type (edited)

Domiciles

Noble family titles can be tiered now with tier = <county/duchy>: give_noble_family_title - Give a noble family title to the <root> character. usage: give_noble_family_title = { name = key/dynamic description tier = <county/duchy> # Optional. Defaults to duchy. article = SOME_TITLE_NAME_ARTICLE government = administrative_government # optional government, default is administrative save_scope_as = <name> } Supported Scopes: characterNomad and adventurer titles are unchanged. NOTE: Interestingly, set_noble_family_title says the title still needs to be duchy-level. There's also add_animate_domicile_building to construct domicile buildings with animation. (edited)

Setting Title Colors with color values

We can set title colors, big for custom map modes where we color baronies (rip the performance with the new amount of baronies though). set_title_color - Changes the title's map color to the specified value. Both named and RGB values are supported. usage: scope:landed_title = { set_title_color = { 255 0 0 } } Supported Scopes: landed_title (edited)
6:03 AM

Liege don't always have to defend vassals?!

set_vassal_wars_are_redirected_to_holder - Flags this title so that the holder will be called to defend their vassals or not according to the given value. This must be their primary title in order for the effect to take place. scope:landed_title = { set_vassal_wars_are_redirected_to_holder = yes/no } Supported Scopes: landed_title
6:07 AM

Situations

Situation pins : set_situation_center_province - Sets a center province for the situation, useful for displaying pins in a specific place. You can now provide a character and a county for catalysts. trigger_situation_catalyst = { catalyst = catalyst_type character = scope:character (optional, the actor who will trigger the catalyst) county = scope:province (optional, identifies the subregion where the catalyst will trigger) }You can trigger a sub_region_catalyst: scope:situation_sub_region = { trigger_sub_region_catalyst = { catalyst = catalyst_type character = scope:character } } You can add/remove realms to situation sub regions: add_character_realm_to_sub_region = <character> + remove_character_realm_from_sub_region = <character> You can add/remove de jure counties too: add_dejure_title_to_sub_region = <title> + remove_dejure_title_from_sub_region = <title> Since participants groups are no longer calculated every day, you can use: scope:ruler = { recalculate_participant_group = scope:situation } (edited)

Barter Goods? - A new currency

Effects

add_barter_goods - adds barter goods to a character Supported Scopes: character remove_barter_goods - removes barter goods from a character Supported Scopes: character Also works as a currency that can be used for casus belli contribution.

situation_top_barter_goods - Get the participating character with the most barter goods in the scoped situation. Input Scopes: situation Output Scopes: character barterer barter_loot
