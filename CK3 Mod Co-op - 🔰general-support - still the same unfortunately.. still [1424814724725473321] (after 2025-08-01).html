Errors when using my svalue that uses integer_range inside a effect block in story's triggered_effect which is stupid af, it suggests that even though it is inside a effect the fact that it is inside a triggered_effect it is treated as a trigger and so no randomization is available! Anyhoo, consider this approach that allows the triggered_effect to enter as long as it's triggers are true and then uses random inside it to simulate the story chance. In operation this is no different than using the story chance because in that scenario the chance has to be met for the triggers to be evaluated, whereas with this approach the triggers have to be true in order for the chance to happen, both approaches give the same effective result wrt the effect actually happening.
