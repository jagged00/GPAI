As for the Reichskrone, you can find out what may be going wrong by checking the portrait modifier for it. (edited)
11:03 AM
gfx/portraits/portrait_modifiers
11:04 AM
Reichskrone is dde_hre_imperial in 01_headgear_base.txt
11:04 AM
From there you see it's applied if portrait_reichskrone_trigger = yes, so we go to that definition
11:05 AM
It's in 00_clothing_triggers.txt, you'll find that a lot of what you need to edit is in there
11:05 AM
Look at the definition, and it's simple: portrait_reichskrone_trigger = { exists = this OR = { has_primary_title = title:e_hre any_equipped_character_artifact = { has_variable = reichskrone } } }
11:06 AM
If you erased e_hre from the game, as most total conversions do, but didn't edit this trigger to remove the reference to title:e_hre, as you're meant to do, then it'll always be true and everyone will wear the crown.
11:07 AM
This is the general practise for debugging. Work from portrait_modifiers. Start by assigning your culture some vanilla gfx and making it work. Then, when it comes to adding new clothes, you'll need to do a bunch of more edits to genes and such.

KeizerHarm | T4N
@maricontrar okay, so sorry I didn't get to it sooner So if you don't have masks then what you need to do is create three files, diffuse, normal, properties. All of them are 1024x1024 .dds files, all of them with mipmaps.
  • yourassetname_diffuse.dds is just your base colour; to make it nicer you can bake AO on top. It's typically compressed with DXT1 in vanilla but not always.
  • yourassetname_normal.dds is your normal texture, always compressed with DXT5. Buuuut it's not a normal normal texture, the channels are swapped. To get from a regular normal map to one CK3 uses, you:
    • copy the green channel to the alpha
    • copy the red channel to green
    • make the alpha channel black
  • yourassetname_properties.dds combines three textures, assigned by channel, compressed with DXT5.
    • red is unused (make it black)
    • green is specular
    • blue is metallic
    • alpha is roughness
So what most of us do is use Gimp for the channel rearranging/recombination. It's got a nice tool: Colours->Components->Decompose. Select RGBA. Then you have a new image with four different greyscale layers corresponding to the channels. There you can do swapping, paste in textures from other sources, etcetera. Finally when done, Colours->Components->Recompose to update the original image again. Then you can close the decomposed one. (edited)

@PiGu | Medieval Arts From what I can see, both holding and special building clearance is set in the NHoldingGraphics entry of the 00_graphics defines file. Specifically, the CLEAR_RADIUS setting sets how much is cleared for holdings and the CLEAR_RADIUS_SPECIAL_BUILDING is for special buildings. NHoldingGraphics = { MAX_ACTIVE_SOUND_EFFECTS = 5 FLAG_MESHES = { "western_flag_01_mesh" } CLEAR_RADIUS = 7.0 # The radius (in map pixels) of a disk that will get cleared around the holding when built. CLEAR_RADIUS_SPECIAL_BUILDING = 7.0 # The radius (in map pixels) of a disk that will get cleared around special buildings when built. If the special building is closer to the holding than (CLEAR_RADIUS + CLEAR_RADIUS_SPECIAL_BUILDING), it won't get cleared. CLEAR_LAYERS = { "tree_low_layer" "tree_medium_layer" "tree_high_layer" } # Which layers to clear around holdings. # Province graphics scale on zoom step PROVINCE_GRAPICS_SCALE_PER_ZOOM_STEP = { 1.0 1.0 1.1 1.2 1.4 1.6 1.8 2.0 0.0 } } So I think you just need to play around with a single custom define file of NHoldingGraphics = { CLEAR_RADIUS_SPECIAL_BUILDING = 7.0 } until you find what works. First of all beautiful model, but 100k is not optimized. It's probably fine for this one model but if you're going to make new ones you want to use a different style. You'd need to share the specific number of tris for each of the main objects to be more helpful, but for example, players are unlikely to ever play at this level of zoom, even if your mod has increased it. There's plenty of details you can cut out here, the main one being the levels of arches in the Colosseum, as well as the windows in the forum's basilicas. Curves are the one thing that most balloons the tris count, and here you have way too many to even be noticed by the player. Same goes for the columns, not every column needs to be clearly modeled, and when you want them you can and should decrease their number, since from other levels of zoom they just become a visual mess. Houses are also larger than the ones in city holdings. That's just a stylistic choice, it's not necessarily a bad thing, but personally I would say try to strike to use the same size for them.
